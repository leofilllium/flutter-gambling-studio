# Как создать MVP гемблинг-игры с нуля

Это руководство проведёт вас через процесс создания мини-слота или рулетки
в `flutter-gambling-studio` от пустой папки до работающего APK.

> **Важно**: Вся разработка ведётся через агентов Claude Code. Вы — режиссёр
> процесса, а агенты пишут математику, дизайн, код и эффекты.

---

## Вариант 1: Путь в один клик (Магия)

Если вы хотите получить готовую игру мгновенно и без лишних вопросов:

```bash
/autocreate
```

Студия сама выберет 1 из 6 архетипов игры (например, "Неоновый 3-барабанный слот"),
сгенерирует `pubspec.yaml`, нарисует SVG, напишет логику барабанов, UI и настроит
проект. Через несколько минут вы сможете запустить `flutter run`.

---

## Вариант 2: Дизайнерский путь (Шаг за шагом)

Если вы хотите контролировать каждый аспект (RTP, волатильность, тематику):

### Шаг 1. Концепция (Brainstorm)
Сначала нужно придумать идею и математическую модель.

```bash
/brainstorm
```
Агент спросит вас о тематике (Викинги? Киберпанк?) и волатильности (Частые но
мелкие выигрыши или редкие джекпоты?). В результате у вас появится файл
`design/gdd/gambling-concept.md`.

### Шаг 2. Разбор на компоненты
```bash
/map-systems
```
Студия разберёт вашу идею на архитектуру классов для Flame. Эта карта поможет
программистам понять масштаб работы.

### Шаг 3. Настройка математики (RTP)
В гемблинге всё начинается с процентов. У слота должен быть RTP (Return to Player)
около 95–97%.
```bash
/design-system rtp-weights
```
На сцену выйдет `rtp-mathematician`. Вы укажете символы, а он рассчитает веса
для честного рандома (`Random.secure()`).

### Шаг 4. Отрисовка графики (SVG)
Дайте команду нарисовать базовые ассеты.
```bash
/generate-asset ui spin-button
/generate-asset symbol вишня
/generate-asset symbol 777
```

### Шаг 5. Оркестрация разработки (Код + VFX)
Когда план, математика и ассеты готовы — зовём команду:
```bash
/team-gambling "Постройте ядро слота по нашему дизайн-документу"
```
Этот навык заставит `slot-programmer` (пишет логику Flutter+Flame) и `juice-artist`
(настраивает тряску камеры, партикли выигрышей) работать вместе.

### Шаг 6. Игровое аудио
Гемблинг без звука — мёртв.
```bash
@sound-designer Настрой звуки барабанов и звон монет через flame_audio.
```

### Шаг 7. Симуляция и контроль качества (QA)
```bash
/balance-check
```
Скрипт прокрутит слот 1,000,000 раз в фоне, чтобы убедиться, что казино (вы)
не останетесь в минусе, а игрок получает свои 95% выигрышей.

### Шаг 8. Релиз
Когда вы можете крутить слот через `flutter run`, пора проверить качество.
```bash
/release-checklist
```
Этот менеджер убедится, что вы не используете нечестный `math.Random()` вместо
криптографического, и что состояние счета не "протекает".

---

🎉 **Готово! Ваш гемблинг MVP собран.**
Вы можете улучшать его, добавляя фичи командой: `/add-feature "добавь Wild символ"`.
