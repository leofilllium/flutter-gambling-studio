## Зачем это нужно

Создавать гемблинг игру в одиночку с ИИ — сложная задача, требующая учета множества нюансов: математический баланс (RTP), juiciness, генераторы случайных чисел (RNG), анимации без задержек и строгая архитектура. Никто не остановит вас от ошибки в расчетах вероятностей или скучного визуального стиля.

**Flutter Gambling Studio** решает эту проблему. Вместо одного универсального помощника вы получаете 10 специализированных агентов, организованных в студийную иерархию гемблинг-разработки. Математики считают RTP, дизайнеры придумывают слоты, джус-художники делают вращения сочными, а программисты реализуют это на движке Flame.

Результат: вы контролируете стратегию разработки и принимаете финальные решения, а виртуальная команда берет на себя рутину: рассчитывает веса символов, строит архитектуру и ловит ошибки логики.

## Иерархия студии

Агенты организованы в уровни (Tier), имитируя профессиональную студию разработки гемблинга:

```
Tier 1 — Директора (Высокоуровневые решения)
  creative-director     

Tier 2 — Специалисты гемблинга (Специфика)
  rtp-mathematician      gambling-game-designer
  slot-programmer        juice-artist          

Tier 3 — Базовые специалисты (Разработка и качество)
  lead-programmer        ui-programmer         
  sound-designer         qa-tester
  release-manager
```

> **Примечание**: Все агенты студии общаются исключительно на **русском языке**. Исключения делаются только для имен переменных, классов, путей к файлам и команд терминала.

## Слэш-команды

Введите `/` в Claude Code для доступа к встроенным процессам пайплайна разработки игры:

**Разработка от начала и до конца:**
`/start` `/brainstorm` `/auto-idea` `/autocreate`

**Архитектура и дизайн:**
`/map-systems` `/design-system` `/prototype`

**Ассеты и контент:**
`/generate-asset` — создание SVG/PNG элементов слота, фишек, символов.

**QA и Баланс:**
`/balance-check` — симуляция 1,000,000 спинов для проверки RTP.
`/release-checklist` — чеклист гемблинг-специфики для безопасного релиза.

**Координация и развитие:**
`/team-gambling` — вызов группы специалистов.
`/continue-project` `/add-feature`

## Быстрый старт

### Требования

- [Flutter SDK](https://docs.flutter.dev/get-started/install) (3.27+)
- [Claude Code](https://docs.anthropic.com/en/docs/claude-code) (`npm install -g @anthropic-ai/claude-code`)

### Настройка и использование

1. **Клонируйте проект и перейдите в папку**:
   ```bash
   git clone https://github.com/leofilllium/flutter-gambling-studio.git
   cd flutter-gambling-studio
   ```

2. **Откройте Claude Code** и запустите сессию:
   ```bash
   claude
   ```

3. **Начните работу с одной из ключевых команд**:
   - `/start` — получить вводную от студии и понять контекст.
   - `/brainstorm` — поштурмить идею совместно с искусственным интеллектом, настраивая RTP-профиль.
   - `/autocreate` — запустить автономный zero-to-playable процесс. Вы лишь наблюдаете, как генерируется игра!

## Архетипы мини-игр

Студия умеет автономно создавать механики на основе 6 архетипов:

| ID | Название | Жанр | Уникальная механика |
|----|----------|------|---------------------|
| A | Неоновый Спин | 3-барабанный слот | Near Miss система, каскадная остановка |
| B | Счастливое Колесо | Колесо фортуны | Сектора с разными выплатами, множители |
| C | Покер Экспресс | Видео-покер | 5 карт, Hold функция, двойная ставка |
| D | Фруктовая Буря | 5-барабанный слот | Каскадные символы (Avalanche), Free Spins |
| E | Скрэтч Делюкс | Скретч-карты | Мгновенные выигрыши, 3 попытки |
| F | Рулетка Неон | Мини-рулетка | Европейская рулетка, внешние/внутренние ставки |

> Попробуйте команду `/auto-idea --archetype A` для быстрой генерации концепта неонового слота.

## Как это работает

**Сотрудничество под управлением пользователя, не автономное выполнение (кроме `/autocreate`).**

Схема работы студии выглядит так:
`Вопрос` → `Варианты` → `Ваше решение` → `Черновик` → `Ободрение`

1. **Спросить перед записью** — Агенты ОБЯЗАНЫ спрашивать «Могу ли я записать это в [путь]?» перед Write/Edit.
2. **Итеративность** — разработка слота разбита на этапы: Дизайн (GDD) → Математика (RTP) → Прототип (Feel/Juice) → Код → Баланс.
3. **Автономные исключения** — такие команды как `/auto-idea` и `/autocreate` созданы для того, чтобы работать полностью без вмешательства. ИИ возьмет всё на себя.

## Кастомизация

Вы можете развивать свою гемблинг студию:

- Добавляйте новые механики к архетипам.
- Правьте `.claude/agents/rtp-mathematician.md`, чтобы усложнить формулы расчета волатильности.
- Настраивайте промпты со словарным запасом в `.claude/docs/coordination-rules.md`.
- Интегрируйте новые навыки в `.claude/skills/`.

## Локализация

Интерфейс студии для общения разработчиков **строго зафиксирован на русском языке**, что позволяет команде ИИ-агентов эффективнее обмениваться документацией с конечным пользователем. Однако исходный код игр (Dart) всегда пишется согласно стандартам (на английском языке).
