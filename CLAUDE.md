# Flutter Gambling Studio — Студия мини-гемблинг игр

> Специализированная студия для создания мини-гемблинг игр (слоты, рулетка, карточные игры)
> на Flutter + Flame через координированных агентов Claude Code.

## Технологический стек

- **Движок**: Flutter 3.27+ / Flame 1.18+
- **Язык**: Dart 3.6+ (null-safe, sealed classes, pattern matching)
- **Специализация**: мини-гемблинг игры (mobile-first)
- **Рендеринг**: Flutter Impeller (iOS/Android), Skia (desktop)

> **Вы — творческий директор и продюсер.** Агенты реализуют вашу идею.
> Запустите `/start` для начала работы.

## Язык общения

**ВСЕ взаимодействия ведутся на русском языке.** Это обязательное требование для всех агентов.

Исключения (остаются на английском): код Dart/Flutter, пути файлов, имена классов, команды CLI.

## Быстрый старт

```
/start             — Ориентация: с чего начать прямо сейчас
/brainstorm        — Интерактивная генерация концепта гемблинг-игры
/auto-idea         — Автономная генерация готовой идеи (без вопросов)
/autocreate        — Zero-to-playable: концепт + ассеты + код за одну команду
```

## Полный путь создания игры

```
Идея → Концепт → Дизайн → Gate → Код → UI Audit → Gate → QA → Gate → Релиз
  │       │         │        │      │      │         │      │       │       │
/start /brain-  /design-  /gate  /team- /ui-     /code- /balance /gate  /release-
       storm    system    check  gambl  audit    review  check   check  checklist
```

## Команды студии (навыки)

### Создание игры (по порядку)

| Команда | Описание | Когда использовать |
|---------|----------|-------------------|
| `/start` | Онбординг и маршрутизация | В начале каждой сессии |
| `/brainstorm [хинт]` | Интерактивный концепт с RTP-профилем | Нет идеи или нужно оформить |
| `/auto-idea` | Автономный концепт (12 архетипов / Уникальный) | Быстрая генерация без вопросов |

| `/auto-idea --archetype [A-L]` | Развернуть конкретный архетип | Есть предпочтение |
| `/autocreate` | Полный конвейер без вопросов | Хочу игру прямо сейчас |

| `/map-systems` | Декомпозиция на системы | После концепта |
| `/design-system [система]` | GDD для одной системы слота | По одной системе за раз |
| `/prototype [механика]` | Прототип juiciness / feel | Перед полной реализацией |

| `/generate-asset [тип] [имя] --png` | Сразу PNG режим через Google Imagen API | Когда нужен растр |
| `/generate-png-asset [описание]` | Мгновенный PNG с удалением фона через Google Imagen | Нужен качественный растр быстро |
| `/generate-png-asset --batch "cherry,bar,seven"` | Batch-генерация нескольких PNG за раз | Генерация всех символов |
| `/generate-png-asset --from-concept` | Генерировать все PNG из symbols[] в rtp-config.json | После дизайна математики |
| `/svg-to-png [путь]` | Конвертация готового SVG в PNG через Google Imagen | Есть SVG, нужен растр |

## Критические правила (gambling integrity)

> Нарушение этих правил блокирует релиз:

1. **RNG**: ТОЛЬКО `Random.secure()` — нет `math.Random()`, нет `Random()`
2. **Stateless Outcomes**: результат спина вычислен ДО анимации
3. **SlotConfig**: все игровые константы только в `slot_config.dart`
4. **RTP диапазон**: 95–97% при 1М симуляций (`/balance-check`)
5. **GameState**: sealed class — нет boolean флагов
6. **Двойной клик**: Spin заблокирован во время спина
7. **No hardcoded probability**: нет `if (rng < 0.1) win!`

## Протокол совместной работы

**Сотрудничество под управлением пользователя, не автономное выполнение.**

Схема: **Вопрос → Варианты → Решение → Черновик → Одобрение**

- Агенты ОБЯЗАНЫ спрашивать «Могу ли я записать это в [путь]?» перед Write/Edit
- Исключение: `/autocreate` и `/auto-idea` работают автономно (это намеренно)

## Anti-Slop UI/UX дизайн

> Студия создаёт **craft-level** UI, который невозможно спутать с AI-шаблоном.
> Запрещены: generic Material Design, фиолетовые градиенты, одинаковые карточки, CircularProgressIndicator.
> Обязательно: кастомные формы, 2+ шрифта, тематические переходы, micro-interactions, 7+ экранов в MVP.
