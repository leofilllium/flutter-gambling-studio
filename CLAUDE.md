# Flutter Game Studio — Студия мини-игр

> Универсальная студия для создания мини-игр любого жанра (гемблинг, пазлы, аркады, физика, карточные)
> на Flutter + Flame через координированных агентов Claude Code.
>
> Для OpenAI Codex используйте `AGENTS.md` и `.codex/` как слой совместимости поверх этих же правил.

## Технологический стек

- **Движок**: Flutter 3.27+ / Flame 1.18+
- **Язык**: Dart 3.6+ (null-safe, sealed classes, pattern matching)
- **Специализация**: мини-игры любого жанра (mobile-first)
- **Рендеринг**: Flutter Impeller (iOS/Android), Skia (desktop)

> **Вы — творческий директор и продюсер.** Агенты реализуют вашу идею.
> Запустите `/start` для начала работы.

## Язык общения

**ВСЕ взаимодействия ведутся на русском языке.** Это обязательное требование для всех агентов.

Исключения (остаются на английском): код Dart/Flutter, пути файлов, имена классов, команды CLI.

## Быстрый старт

```
/start             — Ориентация: с чего начать прямо сейчас
/brainstorm        — Интерактивная генерация концепта игры любого жанра
/auto-idea         — Автономная генерация готовой идеи (без вопросов)
/autocreate        — Zero-to-Production: полная рабочая игра за одну команду
                     (концепт + ассеты + код + тесты + аудит + баланс)
```

## Полный путь создания игры

```
Идея → Концепт → Дизайн → Gate → Код → UI Audit → Runtime → Gate → QA → Gate → Релиз
  │       │         │        │      │      │          │        │      │       │       │
/start /brain-  /design-  /gate  /team- /ui-     /emulator- /code- /balance /gate /release-
       storm    system    check  dev    audit    test       review  check   check checklist
```

## Команды студии (навыки)

### Создание игры (по порядку)

| Команда | Описание | Когда использовать |
|---------|----------|-------------------|
| `/start` | Онбординг и маршрутизация | В начале каждой сессии |
| `/brainstorm [хинт]` | Интерактивный концепт любого жанра | Нет идеи или нужно оформить |
| `/auto-idea` | Автономный концепт (24 архетипа A–X / Уникальный) | Быстрая генерация без вопросов |
| `/auto-idea --list` | Показать все 24 архетипа всех жанров | Выбор архетипа вручную |
| `/auto-idea --archetype [A-X]` | Развернуть конкретный архетип | Есть предпочтение |
| `/autocreate` | Zero-to-Production: полная рабочая игра без вопросов | Хочу полностью рабочую игру |
| `/autocreate --from-concept` | Реализовать сохранённую идею | После `/auto-idea` |
| `/map-systems` | Декомпозиция на системы | После концепта |
| `/design-system [система]` | GDD для одной системы игры | По одной системе за раз |
| `/prototype [механика]` | Прототип juiciness / feel | Перед полной реализацией |
| `/generate-asset [тип] [имя]` | SVG (по умолчанию) или PNG ассеты — спрашивает режим | Перед кодом |
| `/generate-asset [тип] [имя] --png` | Сразу PNG режим через Google Imagen API | Когда нужен растр |
| `/generate-png-asset [описание]` | Мгновенный PNG с удалением фона через Google Imagen | Нужен качественный растр быстро |
| `/generate-png-asset --batch "items"` | Batch-генерация нескольких PNG за раз | Генерация всех ассетов |
| `/generate-png-asset --from-concept` | Генерировать все PNG из концепта | После дизайна |
| `/svg-to-png [путь]` | Конвертация готового SVG в PNG через Google Imagen | Есть SVG, нужен растр |

### Ворота качества (проходить перед каждым переходом)

| Команда | Описание | Когда |
|---------|----------|-------|
| `/gate-check concept` | Готов ли концепт к дизайну? | После brainstorm |
| `/gate-check design` | Готов ли GDD к имплементации? | Перед командой программистов |
| `/gate-check code` | Готов ли код к QA? | После завершения кодирования |
| `/gate-check qa` | Готов ли к релизу? | После всех тестов |

### Ревью и качество

| Команда | Описание |
|---------|----------|
| `/code-review` | Полное ревью кода (архитектура, Flame API, State, тесты) |
| `/ui-audit` | Автоматический аудит UI/UX на anti-slop качество + автофикс |
| `/emulator-test` | Runtime-верификация на ADB/эмуляторе: запуск, скриншоты, визуальный анализ (vision), парсинг logcat, автофикс багов (пустой игровой экран, overflow, red screen, missing assets) |
| `/design-review` | Ревью GDD на полноту и корректность |
| `/balance-check` | Балансировка игры: RTP-симуляция (gambling) или difficulty curve (другие жанры) |
| `/release-checklist` | Финальный GO/NO-GO чеклист перед релизом (агент release-manager) |
| `/release-package` | Упаковка релиза: скрины всех экранов + release APK + `flutter clean` + ZIP в `project_zip/` |

## Критические правила (game integrity)

> Нарушение этих правил блокирует релиз:

### Для ВСЕХ жанров:
1. **GameState**: sealed class — нет boolean флагов
2. **GameConfig**: все игровые константы только в конфиг-файле (`game_config.dart`)
3. **Двойная защита**: основная кнопка действия заблокирована во время анимации
4. **No magic numbers**: нет захардкоженных игровых параметров вне конфига

### Дополнительно для Gambling жанра:
5. **RNG**: ТОЛЬКО `Random.secure()` — нет `math.Random()`, нет `Random()`
6. **Stateless Outcomes**: результат спина вычислен ДО анимации
7. **RTP диапазон**: 95–97% при 1М симуляций (`/balance-check`)
8. **No hardcoded probability**: нет `if (rng < 0.1) win!`

## Протокол совместной работы

**Сотрудничество под управлением пользователя, не автономное выполнение.**

Схема: **Вопрос → Варианты → Решение → Черновик → Одобрение**

- Агенты ОБЯЗАНЫ спрашивать «Могу ли я записать это в [путь]?» перед Write/Edit
- Исключение: `/autocreate` и `/auto-idea` работают автономно (это намеренно)

## Contextual Design (Human-Crafted UI)

> Каждое визуальное решение вытекает из контекста КОНКРЕТНОЙ игры — её темы, настроения, механики.
> Нет единого шаблона. Neon-трапеции для ВСЕ игр — такой же slop, как фиолетовые градиенты.
> Лесной пазл выглядит как лес. Космический слот — как космос. Уютный кликер — как уют.
> Тест: если UI можно перенести на другую игру без изменений — дизайн провалился.
