# Anti-AI-Slop UI/UX Design Rules

> AI-сгенерированные интерфейсы выглядят одинаково: скруглённые карточки, фиолетовые градиенты,
> пустые пространства, generic иконки. Это — "slop". Наша студия создаёт **craft-level** UI,
> который невозможно спутать с шаблоном.

## Что такое AI Slop в UI

Признаки, которые ЗАПРЕЩЕНЫ в наших играх:

| Slop-паттерн | Почему плохо | Наша альтернатива |
|-------------|-------------|-------------------|
| Одинаковые скруглённые карточки везде | Ленивая сетка без иерархии | Разная геометрия для разных функций: скошенные углы для кнопок действий, закруглённые для информации, острые для предупреждений |
| Фиолетовый/синий градиент на всём | Дефолтный AI цвет | Тематическая палитра из GDD: 3 основных + 2 акцентных цвета с обоснованием |
| Симметричная центрированная раскладка | Скучно и нечитаемо | Асимметричная сетка с визуальным весом: важное крупнее и левее |
| Пустые белые/тёмные пространства | Нет атмосферы | Текстурированные фоны, subtle паттерны, ambient анимации, партиклы |
| Generic иконки (Material Icons as-is) | Безликость | Кастомные SVG иконки в стиле игры или стилизованные Material icons |
| Одинаковый размер шрифта везде | Нет иерархии | 4+ уровня типографики с контрастными размерами (12/16/24/40+) |
| Кнопки-прямоугольники со скруглением | Каждое AI-приложение | Shaped кнопки: трапеции, скошенные, с вырезами, с рамкой-анимацией |
| "Flat" дизайн без глубины | Мёртвый интерфейс | Слои: тени, glow, parallax-подобный эффект, glassmorphism (`BackdropFilter`) |
| Стандартные переходы (fade/slide) | Предсказуемо | Тематические транзиции: шторка слота, разрыв карт, вращение рулетки |

## Обязательные принципы Anti-Slop дизайна

### 1. Визуальная Иерархия через Контраст

```
НЕ: все элементы одного размера в ряд
ДА: баланс крупно сверху, барабаны доминируют центр,
    кнопка Spin крупнее остальных в 2x, ставка мелко но читаемо
```

Правило **60-30-10**:
- 60% экрана — основное действие (барабаны / колесо / карты)
- 30% — HUD и управление
- 10% — декоративные элементы и атмосфера

### 2. Тематическая Консистентность

Каждая игра имеет **Design DNA** — набор визуальных мутаций, пронизывающих весь UI:

```dart
// ✅ ПРАВИЛЬНО — DNA пронизывает всё
// Тема "Неон": скошенные углы + glow + моноширинный шрифт
// Тема "Золото": плавные арки + золотая рамка + serif шрифт
// Тема "Киберпанк": глитч-эффекты + резкие углы + зелёный терминал

// ❌ SLOP — Material Dark Theme без модификации
ThemeData.dark() // ЗАПРЕЩЕНО использовать без кастомизации
```

### 3. Микро-взаимодействия на КАЖДОМ элементе

Ни один интерактивный элемент не должен быть "мёртвым":

| Элемент | Hover/Focus | Tap | Disabled |
|---------|------------|-----|----------|
| Кнопка Spin | Glow пульсация + scale 1.02 | Press scale 0.95 + flash | Тусклый + strikethrough эффект |
| Bet +/- | Подсветка стрелки | Число animate + haptic feedback | Серый + locked иконка |
| Баланс | — | Tap показывает историю | — |
| Символ на барабане | — | Tap показывает таблицу выплат | — |
| Меню | Элемент подсвечивается | Ripple + звук | — |

### 4. Не-Прямоугольный Дизайн

```dart
// ❌ SLOP
Container(
  decoration: BoxDecoration(borderRadius: BorderRadius.circular(12)),
  child: Text('SPIN'),
)

// ✅ ANTI-SLOP — кастомная форма
ClipPath(
  clipper: SkewedButtonClipper(skewAngle: 8), // Скошенная трапеция
  child: Container(
    decoration: BoxDecoration(
      gradient: LinearGradient(colors: [theme.accent, theme.accentDark]),
      boxShadow: [BoxShadow(color: theme.accent.withOpacity(0.6), blurRadius: 20)],
    ),
    child: Text('SPIN', style: theme.buttonText),
  ),
)
```

### 5. Анимированные Переходы между Экранами

```dart
// ❌ SLOP
Navigator.push(context, MaterialPageRoute(builder: (_) => GameScreen()));

// ✅ ANTI-SLOP — тематический переход
Navigator.push(context, PageRouteBuilder(
  pageBuilder: (_, __, ___) => GameScreen(),
  transitionsBuilder: (_, animation, __, child) {
    // Эффект "закрывающихся створок слот-машины"
    return SlotDoorTransition(animation: animation, child: child);
  },
  transitionDuration: AnimationConfig.screenTransition,
));
```

### 5.1. Централизованные Анимации
Все тайминги (Durations) и кривые (Curves) ДОЛЖНЫ быть централизованы в конфигурационном файле (например, `lib/theme/animations.dart`). 
ЗАПРЕЩЕНО хардкодить `Duration(milliseconds: 300)` и дефолтные `Curves.easeInOut` внутри компонентов — всё берётся из унифицированного источника.

### 5.2. Продвинутые Визуальные Свойства (Glassmorphism & Depth)
Для оверлеев, диалогов и выделенных панелей ОБЯЗАТЕЛЬНО использование продвинутого рендеринга слоёв через эффект матового стекла (`BackdropFilter` с `ImageFilter.blur`) или параллакса. Головной фон всегда должен быть живым (партиклы/шейдеры), а элементы поверх него должны искажать свет.

### 6. Типографика с Характером

```dart
// ❌ SLOP
TextStyle(fontSize: 16, color: Colors.white)

// ✅ ANTI-SLOP
TextStyle(
  fontFamily: 'Orbitron',      // Тематический шрифт
  fontSize: 16,
  letterSpacing: 2.5,          // Разрядка для gaming feel
  color: theme.textPrimary,
  shadows: [
    Shadow(color: theme.accent.withOpacity(0.8), blurRadius: 8), // Glow
  ],
)
```

**Правило двух шрифтов**:
- Display/числа: тематический (Orbitron, Press Start 2P, Audiowide, Bungee)
- Body/описания: читаемый (Rajdhani, Exo 2, Saira)
- НИКОГДА не использовать только один системный шрифт

### 7. Цветовая Палитра — Не Фиолетовый Градиент

Каждая игра определяет палитру в GDD. Запрещённые "AI-дефолтные" палитры:
- `#7C3AED` → `#2563EB` (фиолетово-синий градиент)
- `#6366F1` → `#8B5CF6` (indigo-violet)
- Любой градиент из 2 близких фиолетовых/синих тонов

Обязательно:
- Минимум 1 "неожиданный" акцентный цвет (жёлтый, красный, зелёный)
- Контраст между фоном и акцентом минимум 7:1
- Цвета имеют СМЫСЛ: красный = опасность/проигрыш, зелёный = выигрыш, золотой = премиум

### 8. Загрузка и Пустые Состояния

```dart
// ❌ SLOP — CircularProgressIndicator
Center(child: CircularProgressIndicator())

// ✅ ANTI-SLOP — тематический загрузчик
SlotLoadingWidget(
  // Мини-барабан крутится как индикатор загрузки
  // или монетка вращается
  // или карты тасуются
)

// ❌ SLOP — пустой белый экран
Container() // Ничего не показать

// ✅ ANTI-SLOP
EmptyStateWidget(
  icon: AnimatedCoinFlip(), // Анимированная монетка
  text: 'Сделай первую ставку!',
  action: PulsatingButton(text: 'ИГРАТЬ'),
)
```

## Обязательные UI Компоненты для каждой игры

Минимальный набор (НЕ опционально):

1. **Splash Screen** — анимированный логотип/название (1-2 сек)
2. **Main Menu** — с атмосферным фоном и анимацией (обязательно с партиклами или живым фоном)
3. **Game Screen** — основной игровой экран с HUD
4. **Paytable Screen** — таблица выплат (свайп или скролл)
5. **Settings Screen** — звук, вибрация, auto-spin настройки
6. **Win Overlay** — 3 уровня (small/big/mega)
7. **Insufficient Funds Dialog** — с использованием эффекта Glassmorphism
8. **Loading State** — тематический, не CircularProgressIndicator
9. **Game Rules/Help Screen** — как играть, правила
10. **Daily Bonus / Retention** — ежедневная награда (мини-рулетка или открытие сундуков)
11. **Leaderboard / Stats** — игровая статистика игрока и топ победителей
12. **Player Profile** — профиль с аватаром и статистикой

## Чеклист Anti-Slop (проверяется в /ui-audit)

- [ ] Ни один Container не использует `BorderRadius.circular()` без кастомного clipper рядом
- [ ] `ThemeData.dark()` или `ThemeData.light()` НЕ используется напрямую — только кастомная тема
- [ ] Минимум 2 кастомных шрифта подключены
- [ ] Кнопка Spin имеет hover + press + disabled состояния с анимацией
- [ ] Переходы между экранами — НЕ стандартные MaterialPageRoute
- [ ] Цветовая палитра содержит минимум 1 тёплый цвет (не всё фиолетово-синее)
- [ ] Числа (баланс, выигрыш) анимируются при изменении, не прыгают
- [ ] Загрузка — кастомный виджет, не CircularProgressIndicator
- [ ] Пустые состояния стилизованы с иллюстрацией/анимацией
- [ ] Нет двух элементов UI с одинаковой формой И размером И цветом на одном экране
- [ ] Каждый интерактивный элемент имеет тактильную обратную связь (scale/glow/sound)
- [ ] Должен быть файл с унифицированными таймингами анимаций (`lib/theme/animations.dart`)
- [ ] Должно применяться искажение фона (Glassmorphism, `BackdropFilter`) для панелей
- [ ] Минимум 10 экранов/оверлеев в приложении (в MVP включены Daily Bonus, Profile, Stats)
