### Ключевые обязанности

#### 1. Расчёт весов символов

Для каждого символа определяете:
- **Вес** (Weight): целое число, чем больше — тем чаще выпадает
- **Частота**: Weight / Sum(all weights)
- **Шанс комбинации x3**: (W/ΣW)³

Пример таблицы для 3-барабанного слота:

```
Символ    | Вес | Частота | Шанс x3   | Множитель | Вклад в RTP
----------|-----|---------|-----------|-----------|------------
Вишня     |  40 | 40%     | 6.40%     | x2        | 12.80%
Лимон     |  30 | 30%     | 2.70%     | x3        | 8.10%
Колокол   |  15 | 15%     | 0.34%     | x10       | 3.38%
Бар       |  10 | 10%     | 0.10%     | x25       | 2.50%
Алмаз     |   4 |  4%     | 0.006%    | x100      | 0.64%
Семёрка   |   1 |  1%     | 0.0001%   | x1000     | 0.10%
ИТОГ RTP  |     |         |           |           | 27.52%
```

> ⚠ Такой RTP нереалистичен — символы нужно балансировать итерационно.

#### 2. Целевая формула RTP

```
RTP = Σ (P(combo_i) × Multiplier_i × Bet)
    = Σ ((W_i / ΣW)^reels × Payout_i)
```

Для целевого RTP 96%:
- Начните с таблицы выплат
- Итерируйте веса через бинарный поиск
- Цель: `|actual_RTP - target_RTP| < 0.1%`

#### 3. Симуляция (Python-скрипт)

Когда нужна точная проверка, генерируйте и запускайте Python-скрипт:

```python
# tools/simulate_rtp.py
import random
from collections import Counter

SYMBOLS = {
    'cherry': {'weight': 40, 'payout': 2},
    'lemon':  {'weight': 30, 'payout': 3},
    'bell':   {'weight': 15, 'payout': 10},
    'bar':    {'weight': 10, 'payout': 25},
    'diamond':{'weight':  4, 'payout': 100},
    'seven':  {'weight':  1, 'payout': 1000},
}
REELS = 3
SPINS = 1_000_000

total_bet, total_win = 0, 0
weights = list(s['weight'] for s in SYMBOLS.values())
names = list(SYMBOLS.keys())

for _ in range(SPINS):
    result = random.choices(names, weights=weights, k=REELS)
    total_bet += 1
    if len(set(result)) == 1:  # все одинаковые
        total_win += SYMBOLS[result[0]]['payout']

print(f"RTP: {total_win/total_bet*100:.2f}%")
print(f"Spins: {SPINS:,}, Won: {total_win:,}")
```

#### 4. Near Miss система

Механика которая делает игру более "захватывающей":
- **Near Miss**: два из трёх барабанов показывают winning символ
- Реализуется через контролируемое вращение последнего барабана
- **ВАЖНО**: Near Miss не должен влиять на RTP — только на анимацию

#### 5. Волатильность

| Волатильность | RTP | Частота побед | Макс. выплата |
|---------------|-----|---------------|---------------|
| Низкая        | 96% | ~35%          | x50           |
| Средняя       | 95% | ~20%          | x200          |
| Высокая       | 94% | ~10%          | x1000         |
