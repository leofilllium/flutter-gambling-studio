### Gambling Release Checklist

Перед любым релизом вы должны проверить проект по следующим пунктам:

#### 1. Архитектура и Состояние
- [ ] **Stateless Spin Result**: Результат спина рассчитывается за 1 такт ДО начала анимации барабанов? (Анимация не должна влиять на результат)
- [ ] **State Leakage**: Нет ли утечек состояния между спинами? (Баланс обновляется только 1 раз за спин)

#### 2. Математика и RNG
- [ ] **Secure RNG**: Используется ли `Random.secure()` во всех местах, где определяется выигрыш? (Никаких `math.Random()`)
- [ ] **RTP Check**: Соответствует ли реализованная таблица выплат той, что в GDD?
- [ ] **Simulated RTP**: Есть ли отчёт от `rtp-mathematician` о симуляции 1M спинов? Успешен ли он?

#### 3. UX и Juiciness
- [ ] **Spin Feedback**: Есть ли мгновенный визуальный и аудио-фидбэк на нажатие кнопки Spin?
- [ ] **Cascade Stop**: Останавливаются ли барабаны каскадом (Reel 1 → Reel 2 → Reel 3), а не все разом?
- [ ] **Big Win Seq**: Есть ли специальная анимация для выигрышей > x20?

#### 4. Платформа
- [ ] **No Errors**: Проходит ли `flutter analyze` без единой ошибки?
- [ ] **No Warnings**: Нет ли критических ворнингов (разрешены только TODOs)?

### Протокол работы

1. Запросите у пользователя команду на запуск проверок.
2. Пройдитесь по кодовой базе (lib/systems/, lib/components/) и проверьте чеклист.
3. Составьте отчёт в `production/session-logs/release-[date].md`.
4. Дайте вердикт: GO или NO-GO.
