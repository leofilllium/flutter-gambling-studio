# `design-system` — Детализация гемблинг-систем

Интерактивно проектирует одну локальную систему слота/рулетки/игры, общаясь с игроком.

## Рабочий процесс

1. **Контекст**: Прочитите `design/gdd/gambling-concept.md`
2. **Тип системы**: Определите о какой системе идет речь:
   - **Математика/Веса**: `rtp-mathematician` (RTP, Payload, Weights)
   - **Механика/Бонусы**: `gambling-game-designer` (Scatter, Free Spins)
   - **Audio/VFX**: `juice-artist` / `sound-designer` (Near Miss, Cascade)

3. **Интерактив (Вопросы)**:
   Пример для "rtp-weights":
   - Спрашиваем список базовых символов.
   - Спрашиваем желаемый множитель для самого редкого (x100? x1000?).
   - Спрашиваем распределение "пустых" спинов (частота проигрышей около 70-80% — стандарт).

4. **Генерация черновика**:
   Запишите в `design/gdd/[system-name].md`.
   Обязательные поля для гемблинг-дизайна:
   - *RTP Impact* (как влияет на общую математику)
   - *Visual Feedback* (какой фидбек у игрока на срабатывание)
   - *Edge Cases* (что делать если баланса не хватает)

5. **Дальнейшие шаги**:
   - `/balance-check` (если меняли веса/выплаты)
   - Вызвать `/team-gambling` для программирования этой системы
