### Ключевые обязанности по жанрам

#### Gambling (слоты, рулетка, карточные игры)

Для слота определяете:
- Количество барабанов (3 или 5), видимые строки (1, 3, 5)
- Линии выплат (1 → 3 → 5 → 9+)
- Специальные символы: Wild (джокер), Scatter (рассеиватель), Bonus

| Символ | Описание | Механика |
|--------|----------|----------|
| **Wild** | Джокер | Заменяет любой символ кроме Scatter |
| **Scatter** | Рассеиватель | Платит в любом месте, не только на линии |
| **Bonus** | Бонусный | 3+ запускают бонусный раунд |
| **Multiplier** | Множитель | Умножает выигрыш (x2, x3, x5) |

**Бонусные механики:**
- Free Spins: триггер 3+ Scatter, 10–15 спинов, множитель x2–x3
- Cascading Reels: выигравшие символы исчезают, новые падают сверху
- Bonus Round: мини-игра выбора из предметов

#### Puzzle (Match-3, Tetris, Push-puzzle)

Для Match-3 определяете:
- Размер сетки (6x6, 8x8, 9x9)
- Типы плиток и специальных элементов (бомба, строчная, цвет)
- Цели уровня (счёт, собери N плиток, дойди до низа)
- Систему звёзд (1-3 за уровень)
- Количество ходов или время

#### Action / Arcade (Runner, Shooter, Breakout)

Для аркад определяете:
- Кривая сложности (скорость роста трудности)
- Систему жизней или HP
- Систему очков и комбо
- Пауэрапы и их балансировку
- Паттерны волн врагов (для шутера)
- Типы блоков и пауэрапов (для Breakout)

#### Physics Games (Pinball, Plinko, Projectile)

Для физических игр определяете:
- Физические параметры: гравитация, трение, упругость
- Параметры целей и препятствий
- Систему очков за точность / расстояние
- Баланс сложности vs случайности

#### Casual / Idle (Clicker, Rhythm)

Для казуальных игр определяете:
- Петля прогрессии: что игрок делает → что получает → куда тратит
- Скорость накопления валюты
- Апгрейды и их стоимость
- Для ритм-игр: BPM, сложность паттернов нот

#### Card / Board (Memory, Quiz)

Для карточных игр определяете:
- Количество карт / вопросов
- Таймер и штрафы за ошибки
- Систему streak бонусов
- Прогрессию сложности

### Структура GDD

Создаёт файл `design/gdd/[система].md` со следующими разделами:

1. **Обзор системы**: что делает, зачем
2. **Правила**: все условия однозначно
3. **Параметры**: числа с диапазонами для тюнинга
4. **Взаимодействие**: с какими системами связана
5. **Визуальные требования**: что нужно от art/UI
6. **Аудио-события**: какие звуки нужно воспроизводить
7. **Edge Cases**: граничные ситуации и их обработка
8. **Критерии приёмки**: как проверить что система работает
